
## What I found (live checks against www.scarpers.co.uk)

The GSC report has 3 fixable technical causes. The rest ("crawled, not indexed", "discovered, not indexed") is almost always Google being slow on a small, new site â€” not something we fix with code today.

### Root causes

1. **Homepage `/` has no canonical at all.**
   - `index.html` only has a comment where canonical should be â€” canonicals are injected client-side by `MarketingPageLayout`.
   - `src/pages/Landing.tsx` does **not** use `MarketingPageLayout`, so `/` ships with zero `<link rel="canonical">`.
   - This is the **"Duplicate without user-selected canonical"** finding.

2. **`/coach` vs `/coach/claire-rayners` are the same page with conflicting signals.**
   - Both routes render `CoachClaire`, which sets `canonicalPath="/coach"`.
   - But `sitemap.xml` (and `scripts/generate-sitemap.ts`) lists `/coach/claire-rayners`.
   - Google crawls `/coach/claire-rayners` from the sitemap, sees a canonical pointing to `/coach`, and reports it as **"Page with redirect"** (canonical-redirect). Meanwhile `/coach` isn't in the sitemap.

3. **`scarpers.co.uk` (apex) serves 200, not a redirect to `www`.**
   - `curl -I https://scarpers.co.uk/` â†’ `HTTP/2 200` (same content as www).
   - Same for `scarpers.lovable.app`. Three identical hostnames + JS-only canonical = perfect storm for duplicate-content classification.

## Plan

### 1. Add static canonical + per-route canonical to the homepage *(fixes finding #1)*

- Add `<link rel="canonical" href="https://www.scarpers.co.uk/" />` to `index.html` as the sitewide default (cheap fallback for non-JS crawlers and social previews).
- Update `MarketingPageLayout`'s cleanup so it restores any pre-existing canonical instead of leaving the last route's value behind (it already captures `prevCanonical`, just verify behavior with the new static tag).
- Wrap `src/pages/Landing.tsx`'s SEO bits in a small `useEffect` (or pull it into `MarketingPageLayout`-style logic) that re-asserts `canonical = https://www.scarpers.co.uk/` on mount. Doing it via effect avoids touching the Landing visual layout.

### 2. Make `/coach` and `/coach/claire-rayners` consistent *(fixes finding #2 â€” the "redirect error")*

### 3. Redirect apex â†’ www at the edge *(fixes finding #3)*
